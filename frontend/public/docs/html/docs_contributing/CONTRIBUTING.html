Contributing to QuantDesk

Thank you for your interest in contributing to QuantDesk! We welcome contributions from the community and appreciate your help in making QuantDesk the premier Bloomberg Terminal for Crypto.

ü§ù How to Contribute

1. Report Issues

  • Use GitHub Issues to report bugs
  • Include detailed steps to reproduce the issue
  • Provide system information and error logs

2. Suggest Features

  • Use GitHub Discussions for feature requests
  • Describe the use case and expected behavior
  • Consider the impact on existing users

3. Submit Code

  • Fork the repository
  • Create a feature branch
  • Make your changes
  • Submit a pull request

üöÄ Getting Started

Prerequisites

  • Node.js 18+
  • Git
  • Solana CLI tools
  • Basic knowledge of TypeScript/JavaScript

Development Setup

  1. Fork and Clone
git clone https://github.com/your-username/quantdesk.git
cd quantdesk
  1. Install Dependencies
# Backend
cd backend
npm install

# Frontend
cd ../frontend
npm install
  1. Environment Setup
# Copy environment files
cp backend/.env.example backend/.env
cp frontend/.env.example frontend/.env
  1. Start Development
# Backend
cd backend
npm run dev

# Frontend (new terminal)
cd frontend
npm run dev

üìã Contribution Guidelines

Code Style

  • TypeScript: Use TypeScript for all new code
  • ESLint: Follow our ESLint configuration
  • Prettier: Use Prettier for code formatting
  • Naming: Use descriptive variable and function names

Commit Messages

Use conventional commit format:

type(scope): description

feat(api): add new order endpoint
fix(ui): resolve wallet connection issue
docs(readme): update installation guide

Pull Request Process

  1. Create Feature Branch
git checkout -b feature/your-feature-name
  1. Make Changes
  2. Write clean, well-documented code
  3. Add tests for new functionality

  4. Update documentation as needed

  5. Test Your Changes

# Backend tests
cd backend
npm test

# Frontend tests
cd ../frontend
npm test
  1. Submit Pull Request
  2. Provide clear description of changes
  3. Link to related issues
  4. Include screenshots for UI changes

üîç Code Review Process

What We Look For

  1. Functionality
  2. Does the code work as intended?
  3. Are edge cases handled properly?

  4. Is error handling implemented?

  5. Code Quality

  6. Is the code readable and maintainable?
  7. Are best practices followed?

  8. Is the code properly documented?

  9. Testing

  10. Are tests included for new functionality?
  11. Do existing tests still pass?

  12. Is test coverage adequate?

  13. Security

  14. Are there any security vulnerabilities?
  15. Is input validation implemented?
  16. Are sensitive data handled properly?

Review Timeline

  • Initial Review: Within 48 hours
  • Follow-up Reviews: Within 24 hours
  • Final Approval: Within 1 week

üêõ Bug Reports

Before Reporting

  1. Check Existing Issues: Search for similar issues
  2. Test Latest Version: Ensure you're using the latest code
  3. Gather Information: Collect relevant details

Bug Report Template

**Bug Description**
A clear description of the bug.

**Steps to Reproduce**
1. Go to '...'
2. Click on '....'
3. Scroll down to '....'
4. See error

**Expected Behavior**
What you expected to happen.

**Actual Behavior**
What actually happened.

**Environment**
- OS: [e.g. Windows 10]
- Browser: [e.g. Chrome 91]
- Node.js: [e.g. 18.0.0]
- QuantDesk Version: [e.g. 1.0.0]

**Additional Context**
Any other context about the problem.
